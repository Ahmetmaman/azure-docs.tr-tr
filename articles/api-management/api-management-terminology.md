# <a name="terminology"></a>Terminoloji

Bu makalede, API Yönetimi (APIM) özgü terimler için tanımlar verir.

## <a name="term-definitions"></a>Terim tanımları

* **Arka uç API** - API'nizi ve işlemlerini uygulayan bir HTTP hizmeti. 
* **Frontend API**/**APIM API** - Bir APIM API API'leri barındırmaz, arka uç API'sine dokunmadan ön cepheyi ihtiyaçlarınıza göre özelleştirmek için API'leriniz için cepheler oluşturur. Daha fazla bilgi için içe [aktarVe bir API yayımlayın.](import-and-publish.md)
* **APIM ürünü** - bir ürün, bir veya daha fazla API'nin yanı sıra kullanım kotası ve kullanım koşulları içerir. Bir dizi API ekleyebilir ve geliştirici portalı aracılığıyla geliştiricilere sunabilirsiniz. Daha fazla bilgi için [bkz.](api-management-howto-add-products.md)
* **APIM API işlemi** - Her APIM API geliştiriciler için kullanılabilir bir dizi işlemi temsil eder. Her APIM API, API'yi uygulayan arka uç hizmetine ve arka uç hizmeti tarafından uygulanan işlemlere yönelik işlem haritasına bir başvuru içerir. Daha fazla bilgi için [Mock API yanıtları'na](mock-api-responses.md)bakın.
* **Sürüm** - Bazen bazı kullanıcılara yeni veya farklı API özellikleri yayınlamak isterken, diğerleri şu anda kendileri için çalışan API'ye bağlı kalmak ister. Daha fazla bilgi için [bkz.](api-management-get-started-publish-versions.md)
* **Revizyon** - API'niz kullanıma hazır olduğunda ve geliştiriciler tarafından kullanılmaya başladığında, genellikle bu API'de değişiklik yapmaya ve aynı zamanda API'nizi arayanları rahatsız etmemeye dikkat etniz gerekir. Yaptığınız değişiklikleri geliştiricilere bildirmeniz de yararlıdır. Daha fazla bilgi için [bkz.](api-management-get-started-revise-api.md)
* **Geliştirici portalı** - Müşterileriniz (geliştiriciler) API'larınıza erişmek için Geliştirici portalını kullanmalıdır. Geliştirici portalı özelleştirilebilir. Daha fazla bilgi için Geliştirici [portalını Özelleştir'e](api-management-customize-styles.md)bakın.
