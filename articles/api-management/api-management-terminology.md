# <a name="terminology"></a>Terminoloji

Bu makalede, API Management (APIM) için belirli koşulların tanımlarını sağlar.

## <a name="term-definitions"></a>Terim tanımları

* **Arka uç API** -bir HTTP hizmeti, API'nizi ve işlemlerini uygular. 
* **Ön uç API**/**APIM API'si** -API'leri bir APIM API'si barındırmaz, API arka dokunmadan ihtiyaçlarınıza göre cepheyi özelleştirmek amacıyla Apı'lerinizi sona cepheleri oluşturur. Daha fazla bilgi için [içeri aktarma ve API yayımlama](import-and-publish.md).
* **APIM ürün** -ürünler bir veya daha fazla API yanı sıra kullanım kotası ve kullanım koşullarını içerir. Bir dizi API ekleyebilir ve bunları Geliştirici Portalı aracılığıyla geliştiricilere sunabilirsiniz. Daha fazla bilgi için [oluşturun ve bir ürünü yayımlama](api-management-howto-add-products.md).
* **APIM API'si işlemi** -her APIM API'si, geliştiricilere sunulan bir işlemler kümesini temsil eder. Her APIM API'si API'yi uygulayan arka uç hizmetine başvuru ve bunun işlemleri arka uç hizmeti tarafından uygulanan işlemlere içerir. Daha fazla bilgi için [sahte API yanıtları](mock-api-responses.md).
* **Sürüm** - bazen yeni yayımlamak istediğiniz veya diğerleri şu anda takımınıza API'si ile kullanmayı tercih ederken bazı kullanıcılar için farklı bir API özellikleri. Daha fazla bilgi için [API'NİZİN birden fazla sürümünü yayımlama](api-management-get-started-publish-versions.md).
* **Düzeltme** - API'nizi gitmek için hazır ve bu API için ve aynı zamanda API'nizi arayanlar kesintiye değişiklikler yaparken dikkatli gereken geliştiriciler tarafından genellikle, kullanılacak başlatır. Yaptığınız değişiklikleri geliştiricilere bildirmeniz de yararlıdır. Daha fazla bilgi için [düzeltmeleri kullanma](api-management-get-started-revise-api.md).
* **Geliştirici Portalı** -Geliştirici Portalı, API'lere müşterilerinizin (geliştiriciler) kullanmanız gerekir. Geliştirici Portalı özelleştirilebilir. Daha fazla bilgi için [Geliştirici portalını özelleştirme](api-management-customize-styles.md).
