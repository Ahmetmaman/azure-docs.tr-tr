# <a name="azure-sql-database-features"></a>Azure SQL Veritabanı Özellikleri

Azure SQL Veritabanı hizmeti, SQL Server'ın en son kararlı sürümüyle ortak bir kod tabanını paylaşır. Standart SQL dili, sorgu işleme ve veritabanı yönetimi özelliklerinin çoğu SQL Server ve Azure SQL Veritabanı'nda aynıdır. SQL Server'da ve Azure SQL Veritabanı'nın tüm tatlarında yaygın olan özellikler şunlardır:
  - Dil özellikleri - [Akış dili anahtar kelimelerin kontrolü](https://docs.microsoft.com/sql/t-sql/language-elements/control-of-flow), [İmleçler](https://docs.microsoft.com/sql/t-sql/language-elements/cursors-transact-sql), [Veri türleri](https://docs.microsoft.com/sql/t-sql/data-types/data-types-transact-sql), [DML ifadeleri](https://docs.microsoft.com/sql/t-sql/queries/queries), [Yüklemler](https://docs.microsoft.com/sql/t-sql/queries/predicates), [Sıra numaraları](https://docs.microsoft.com/sql/relational-databases/sequence-numbers/sequence-numbers), [Depolanan yordamlar](https://docs.microsoft.com/sql/relational-databases/stored-procedures/stored-procedures-database-engine), ve [Değişkenler](https://docs.microsoft.com/sql/t-sql/language-elements/variables-transact-sql).
  - Veritabanı özellikleri - [Otomatik ayar (plan zorlama)](https://docs.microsoft.com/sql/relational-databases/automatic-tuning/automatic-tuning), [İzleme değiştir](https://docs.microsoft.com/sql/relational-databases/track-changes/about-change-tracking-sql-server), [Veritabanı harmanlama](https://docs.microsoft.com/sql/relational-databases/collations/set-or-change-the-database-collation), [Dahilli veritabanları](https://docs.microsoft.com/sql/relational-databases/databases/contained-databases), [Dahilli kullanıcılar](https://docs.microsoft.com/sql/relational-databases/security/contained-database-users-making-your-database-portable), Veri [sıkıştırma](https://docs.microsoft.com/sql/relational-databases/data-compression/data-compression), Veritabanı [yapılandırma ayarları](https://docs.microsoft.com/sql/t-sql/statements/alter-database-scoped-configuration-transact-sql), Online [dizin işlemleri](https://docs.microsoft.com/sql/relational-databases/indexes/perform-index-operations-online), [Bölümleme](https://docs.microsoft.com/sql/relational-databases/partitions/partitioned-tables-and-indexes), ve [Zamansal tablolar](https://docs.microsoft.com/sql/relational-databases/tables/temporal-tables) ( başlangıç kılavuzu[başlarken bakınız).](sql-database-temporal-tables.md)
  - Güvenlik özellikleri - [Uygulama rolleri](https://docs.microsoft.com/sql/relational-databases/security/authentication-access/application-roles), Dinamik [veri maskeleme](https://docs.microsoft.com/sql/relational-databases/security/dynamic-data-masking) [(başlangıç kılavuzu alma bakınız](sql-database-dynamic-data-masking-get-started.md)), [Satır Düzey Güvenliği](https://docs.microsoft.com/sql/relational-databases/security/row-level-security), ve Tehdit algılama - tek veritabanı ve [elastik havuzları](sql-database-threat-detection.md) ve yönetilen [örnek](sql-database-managed-instance-threat-detection.md)için kılavuzları başlarken bakın.
  - Çok model yetenekleri - [Grafik işleme](https://docs.microsoft.com/sql/relational-databases/graphs/sql-graph-overview), [JSON veri](https://docs.microsoft.com/sql/relational-databases/json/json-data-sql-server) ( başlangıç kılavuzu, [OPENXML](https://docs.microsoft.com/sql/t-sql/functions/openxml-transact-sql), [Mekansal](https://docs.microsoft.com/sql/relational-databases/spatial/spatial-data-sql-server), [OPENJSON](https://docs.microsoft.com/sql/t-sql/functions/openjson-transact-sql)ve [XML dizinleri](https://docs.microsoft.com/sql/t-sql/statements/create-xml-index-transact-sql)[başlarken bakın](sql-database-json-features.md).

Azure SQL Veritabanı veritabanlarınızı yönetir ve yüksek kullanılabilirliklerini garanti eder. Yüksek kullanılabilirliği etkileyebilecek veya PaaS dünyasında kullanılamayacak bazı özelliklerin Azure SQL Veritabanı'nda sınırlı işlevleri vardır. Ayrıca, bazı veritabanı özellikleri oluşturduğunuz Azure SQL Veritabanı türüne bağlıdır. Bu özellikler aşağıdaki tablolarda açıklanmıştır. Azure SQL Veritabanı ile, yönetilen bir [örneğin](sql-database-managed-instance.md)parçası olarak , tek bir veritabanı olarak veya esnek bir havuzun parçası olarak bir veritabanı oluşturabilirsiniz. Farklar hakkında daha fazla ayrıntıya ihtiyacınız varsa, bunları [Tek veritabanı ve Elastik havuzlar](sql-database-transact-sql-information.md) veya [Yönetilen Örnek](sql-database-managed-instance-transact-sql-information.md)için ayrı sayfalarda bulabilirsiniz.
