# <a name="what-is-azure-api-for-fhirreg"></a>FHıR için Azure API nedir &reg; ?

FHıR için Azure API, bulutta bir yönetilen hizmet olarak platform (PaaS) ile desteklenen hızlı sağlık birlikte çalışabilirlik kaynakları (FHıR®) API 'Leri aracılığıyla verilerin hızla değişimine izin verebilir. Durum verileriyle çalışan herkesin, bulutta korunan [sağlık bilgilerini alma](https://www.hhs.gov/answers/hipaa/what-is-phi/index.html) , yönetme ve kalıcı hale getirme işlemlerini kolaylaştırır: 

- Dakikalar içinde bulutta sağlanan yönetilen FHıR hizmeti 
- Veri erişimi için Azure 'da kurumsal düzeyde, FHıR® tabanlı uç nokta ve FHıR® biçiminde depolama
- Yüksek performans, düşük gecikme süresi
- Uyumlu bir bulut ortamında korunan durum verilerinin (FI) güvenli yönetimi
- Mobil ve Web uygulamaları için FHıR 'de akıllı
- Rol tabanlı Access Control (RBAC) ile kendi verilerinizi ölçeklendirerek denetleyin
- Her veri deposu içinde erişim, oluşturma, değiştirme ve okuma için günlük izlemeyi denetleme

FHıR için Azure API 'SI, bulutun esnek ölçeğinden yararlanmak üzere bir FHıR hizmetini yalnızca dakikalar içinde oluşturup dağıtmanıza olanak tanır.  Yalnızca ihtiyacınız olan aktarım hızı ve depolama alanı için ödeme yaparsınız. FHıR için Azure API 'YI destekleyen Azure Hizmetleri, yönettiğiniz veri kümelerinin ne olduğuna bakılmaksızın hızlı performans için tasarlanmıştır.

FHıR API 'SI ve uyumlu veri deposu, FHıR API 'Leri kullanan tüm sistemleri güvenli bir şekilde bağlamanıza ve bunlarla etkileşime giresağlar.  Microsoft, PaaS sunumunda işlemler, bakım, güncelleştirmeler ve uyumluluk gereksinimlerini alır, böylece kendi işletimsel ve Geliştirme kaynaklarınızı boşaltabilirsiniz.

## <a name="leveraging-the-power-of-your-data-with-fhir"></a>FHıR ile verilerinizin gücünden yararlanın

Sağlık sektörü, sistem durumu verilerini [hızlı bir şekilde &reg; ](https://hl7.org/fhir) (hızlı sağlık birlikte çalışabilirlik kaynakları) gelişen standart bir standarda dönüştürüyor. FHAR, FHıR kullanan tüm sistemlerin birlikte çalışmasını sağlayan standartlaştırılmış semantik ve veri değişimine sahip sağlam, genişletilebilir bir veri modeli sağlar.  Verilerinizi FHıR 'ye dönüştürmek, elektronik sistem durumu kayıt sistemleri veya araştırma veritabanları gibi mevcut veri kaynaklarını hızlıca bağlamanıza olanak tanır. FHıR, mobil ve Web geliştirme için modern uygulamalarda verilerin hızlı değişimini de mümkün. En önemlisi, FHıR veri alımı basitleştirecek ve analiz ve makine öğrenimi araçlarıyla geliştirmeyi hızlandırmaya devam edebilir.  

### <a name="securely-manage-health-data-in-the-cloud"></a>Bulutta sistem durumu verilerini güvenli bir şekilde yönetin

FHıR için Azure API 'SI, HL7 FHıR belirtimine göre tutarlı, sorunsuz, FHıR API 'Leri aracılığıyla veri değişimine izin verir. Azure 'da yönetilen bir PaaS teklifiyle desteklenen Ayrıca, yerel FHıR biçimindeki korunan sağlık bilgileri (FI) verilerinin yönetimi ve depolanması için ölçeklenebilir ve güvenli bir ortam sağlar.  

### <a name="free-up-your-resources-to-innovate"></a>Kaynaklarınızı yenilik yapın 'e boşaltın

Kendi FHıR hizmetinizi oluşturan ve çalıştıran kaynakları yatırmaya, ancak Microsoft 'un Azure API 'SI ile FHıR için, Microsoft 'un işletimsel ve Geliştirme kaynaklarınızı boşaltmanızı sağlayan işlemler, bakım, güncelleştirmeler ve uyumluluk gereksinimlerinin iş yükünü alır.

### <a name="enable-interoperability-with-fhir"></a>FHAR ile birlikte çalışabilirliği etkinleştir

FHıR için Azure API 'yi kullanmak, okuma, yazma, arama ve diğer işlevler için FHıR API 'Lerinden yararlanan herhangi bir sistemle bağlanmanıza olanak sağlar.  Elektronik sistem durumu kayıtlarından, klinician ve hasta panolarından, uzaktan izleme programlarında veya FHıR API 'Lerine sahip sisteminizin dışındaki veritabanlarıyla makine öğrenimini birleştirmek, normalleştirmek ve uygulamak için güçlü bir araç olarak kullanılabilir.

### <a name="control-data-access-at-scale"></a>Ölçekte veri erişimini denetleme

Verilerinizi kontrol edersiniz. Rol tabanlı Access Control (RBAC), verilerinizin nasıl depolandığını ve erişildiğini yönetmenizi sağlar.  Artırılmış güvenlik sağlar ve yönetim iş yükünü azaltarak, ortamınız için oluşturduğunuz rol tanımlarına göre oluşturduğunuz veri kümelerine kimlerin erişebileceğini belirlersiniz.  

### <a name="audit-logs-and-tracking"></a>Denetim günlükleri ve izleme 

Verilerinizin yerleşik denetim günlükleriyle nerede gittiğini hızlıca izleyin. Her veri deposundaki erişimi, oluşturmayı, değişikliği ve okumaları izleyin.

### <a name="secure-your-data"></a>Verilerinizin güvenliğini sağlama

Paralel güvenlik zekasından PHI koruyun.  Verileriniz API örneği başına benzersiz bir veritabanına ayrılır ve çok bölgeli yük devretme ile korunur. FHıR için Azure API 'SI, verileriniz için katmanlı, derinlemesine bir savunma ve Gelişmiş tehdit koruması uygular.  

## <a name="applications-for-a-fhir-service"></a>FHıR hizmeti için uygulamalar

FHıR sunucuları, sistem durumu verilerinin birlikte çalışabilirliğine yönelik temel araçlardır.  FHıR için Azure API 'si, hızlı bir şekilde oluşturabileceğiniz, dağıtabileceğiniz ve kullanmaya başlayabileceğiniz bir API ve hizmet olarak tasarlanmıştır.  FHıR standardı sağlık 'de genişledikçe, kullanım örnekleri büyümeye devam eder, ancak FHıR için Azure API 'sinin yararlı olduğu bazı ilk müşteri uygulamaları aşağıda verilmiştir: 

- **Başlangıç/IoT ve uygulama geliştirme:**  Hasta veya sağlayıcı merkezli uygulama (mobil veya Web) geliştiren müşteriler, tam olarak yönetilen bir arka uç hizmeti olarak FHıR için Azure API 'den yararlanabilir. FHıR için Azure API 'SI, müşterilerin veri yönetmesi ve sistem durumu verileri için tasarlanan güvenli bir bulut ortamında veri alışverişi yapmasına, FHıR uygulama kılavuzlarından akıllı bir şekilde yararlanmasına ve tüm sağlayıcı sistemleri tarafından kullanılması için teknolojilerin sağlanmasına olanak tanıyan değerli bir kaynak sağlar (örneğin, çoğu EHRs, FHıR okuma API 'Lerini etkinleştirdi).   
- **Sağlık ekosistemleri:**  Bu, birçok klinik ayarlarda birincil ' Truth ' kaynağı olarak var olsa da, sağlayıcıların bir diğerine bağlı olmayan birden çok veritabanına sahip olması veya verileri farklı biçimlerde depolarken yaygın olmayan bir durumdur.  Bu sistemlerin üstünde yer alan bir hizmet olarak FHıR için Azure API 'sinin kullanılmasıyla, verileri FHıR biçiminde standartlaşmanız gerekir.  Bu, tutarlı bir veri biçimiyle birden çok sistemde veri değişimini etkinleştirmeye yardımcı olur. 

- **Araştırma:** Sağlık araştırmacıları, genel olarak FHıR standardını bulur ve FHıR 'nin Azure API 'SI, ortak bir FHıR veri modeli etrafında verileri normalleştirir ve makine öğrenimi ve veri paylaşımı için iş yükünü azaltır.
FHıR için Azure API 'SI aracılığıyla verilerin alışverişi, veri akışını denetlemeye yardımcı olan ve hangi veri türlerine erişebilen denetim günlükleri ve erişim denetimleri sağlar. 

## <a name="fhir-from-microsoft"></a>Microsoft 'tan FHAR

Microsoft 'un FHıR özellikleri iki yapılandırmada sunulmaktadır:

* Azure 'da Azure API 'SI: Azure 'da, Azure portal kolayca sağlanmış ve Microsoft tarafından yönetilen bir PaaS teklifi.
* Azure için FHıR sunucusu – Azure aboneliğinize dağıtılabilecek açık kaynaklı bir proje, ' de GitHub ' da kullanılabilir https://github.com/Microsoft/fhir-server .

FHıR sunucusunu genişletmeyi veya özelleştirmeyi gerektiren veya veritabanı gibi temel hizmetlere erişmek için gerekli olan kullanım örnekleri için, geliştiriciler, Azure için açık kaynaklı FHıR sunucusunu seçmelidir.   Kalıcı verilere yalnızca FHıR API 'SI aracılığıyla erişilmesi gereken bir çift anahtar, üretime hazır FHıR API 'SI ve arka uç hizmeti uygulanması için, geliştiriciler FHıR için Azure API 'sini seçmelidir

## <a name="azure-iot-connector-for-fhir-preview"></a>FHıR için Azure IoT Bağlayıcısı (Önizleme)

FHIR için Azure IoT Bağlayıcısı *, Azure API 'nin, tıp 'ler (IoMT) cihazlarından veri alma özelliğini sağlayan, isteğe bağlı bir Azure API özelliğidir. Tıp interneti, ağ üzerinden BT sistemleri ile veri & elde eden verileri yakalayan ve değiş tokuş eden bir IoT cihazları kategorisidir. Bazı ıomt cihazlarına örnek olarak, uygunluk ve klinik wearables, izleme algılayıcıları, etkinlik izleyicileri, bakım noktaları ve hatta akıllı bir Pill sayılabilir. FHıR için Azure IoT Bağlayıcısı özelliği, Azure API 'ye, ölçeklenebilir, güvenli ve uyumlu bir şekilde Azure API 'sine veri almak üzere bir hizmeti hızla ayarlamanıza olanak sağlar.

FHıR için Azure IoT Bağlayıcısı, bir ıomt cihazı tarafından gönderilen JSON tabanlı iletileri kabul edebilir. Bu veriler ilk olarak uygun FHıR tabanlı [izleme kaynaklarına dönüştürülür](https://www.hl7.org/fhir/observation.html) ve ardından fhır için Azure API 'si ile kalıcı hale getirilir. Veri dönüştürme mantığı, ileti şemanıza ve FHıR gereksinimlerinize göre yapılandırdığınız bir eşleme şablonları çifti aracılığıyla tanımlanır. Cihaz verileri, FHıR için doğrudan Azure IoT Bağlayıcısı 'na veya diğer Azure IoT çözümleriyle ([azure IoT Hub](https://docs.microsoft.com/azure/iot-hub/) ve [Azure IoT Central](https://docs.microsoft.com/azure/iot-central/)) sorunsuz bir şekilde kullanılabilir. FHıR için Azure IoT Bağlayıcısı, Azure IoT çözümlerinin fiziksel cihazların sağlamasını ve bakımını yönetmesine izin verirken güvenli bir veri işlem hattı sağlar.

### <a name="applications-of-azure-iot-connector-for-fhir-preview"></a>FHıR için Azure IoT bağlayıcısının uygulamaları (Önizleme)

IoMT cihazlarının kullanımı, sağlık 'lerde hızlı bir şekilde genişliyor ve FHıR için Azure IoT Bağlayıcısı, birden çok cihaz verisini güvenlik ve uyumluluk ile FHAR için Azure API 'sine getirme konumunu bağlamak üzere tasarlanmıştır. IoMT verilerinin bir FHıR sunucusuna getirilmesi, bütünsel veri öngörüleri ve yenilikçi klinik iş akışları sunar. FHıR için Azure IoT Bağlayıcısı için bazı yaygın senaryolar şunlardır:
- **Uzaktan hasta izleme/TeleHealth:** Uzaktan hasta izleme, sistem durumu verilerinin geleneksel sağlık ayarları dışında toplanbilmesini sağlar. Sağlık kurumları, uzak cihazlar tarafından oluşturulan sistem durumu verilerini FHıR için Azure API 'sine getirmek üzere FHıR için Azure IoT bağlayıcısını kullanabilir. Bu veriler, hastalar sistem durumunu yakından izlemek, hastaların bakım planına yönelik uygunluğunu izlemek ve kişiselleştirilmiş bir bakım sağlamak için kullanılabilir.
- **Araştırma ve Yaşam Bilimleri:** Klinik denemeler, deneme verilerini yakalamak için biyografisi sensör, wearables ve mobil uygulamalar gibi ıomt cihazlarını hızla benimsemektedir. Bu denemeler, FHıR için Azure IoT bağlayıcısını güvenli, verimli ve etkili bir şekilde Azure API 'sine aktarmak için kullanabilir. FHıR için Azure API 'sindeki bir kez deneme verileri, deneme verilerinin gerçek zamanlı analizini çalıştırmak için kullanılabilir.
- **Gelişmiş analiz:** IoMT cihazları yüksek bir hızda büyük hacimli ve çeşitli veriler sunarak, bu sayede makine öğrenimi modelleriniz için eğitim ve test verilerini sunmaya yönelik harika bir uyum sağlayabilir. FHıR için Azure IoT Bağlayıcısı, düşük gecikme süresine sahip çok çeşitli veri sıklığıyla, esnek veri şemasıyla ve bulut ölçeklendirmesiyle çalışacak şekilde kendiliğinden oluşturulmuştur. Bu öznitelikler, Azure IoT bağlayıcısını, gelişmiş analiz gereksinimleriniz için cihaz verilerini yakalamaya yönelik harika bir seçim yapar.
- **Akıllı hastaneler/Klincs:** Günümüzde akıllı hastaneler ve klinikler, birbirine bağlı dijital varlıklar için bir altyapı ayarlıyor. FHıR için Azure IoT Bağlayıcısı, bu bağlı bileşenlerden verileri yakalamak ve bütünleştirmek için kullanılabilir. Bu tür veri kümesinden eyleme dönüştürülebilir Öngörüler, daha iyi hasta ve operasyonel verimliliği sağlar.
